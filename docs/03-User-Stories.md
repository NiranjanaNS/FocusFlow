# 1. Introduction

User stories describe the application from the user's perspective. They capture what users want to accomplish and why those features are valuable.

Each story follows the format:

> **As a _[user]_, I want _[goal]_ so that _[benefit]_.**

These stories help guide development and ensure the application solves real user problems rather than simply implementing technical features.

# 2. User Personas

### Goals

- Organize assignments
- Track deadlines
- Stay focused while studying
- Build consistent study habits

### Challenges

- Forgetting assignments
- Poor time management
- Difficulty staying focused

# 3. MVP User Stories

## Epic 1 – Task Management

### US-001 Create Task

**As a user, I want to create a task so that I can remember work that needs to be completed.**

Priority: High

---

### US-002 Edit Task

**As a user, I want to edit an existing task so that I can keep my task information up to date.**

Priority: High

---

### US-003 Delete Task

**As a user, I want to delete unnecessary tasks so that my task list remains organized.**

Priority: High

---

### US-004 Complete Task

**As a user, I want to mark tasks as completed so that I can track my progress.**

Priority: High

---

### US-005 Mark Task as Pending

**As a user, I want to move a completed task back to pending so that I can continue unfinished work.**

Priority: Medium

---

### US-006 Local Storage

**As a user, I want my tasks to remain available after refreshing the page so that I do not lose my work.**

Priority: High

---

### US-007 Dashboard

**As a user, I want to see task statistics so that I can quickly understand my current progress.**

Priority: High

---

### US-008 Search Tasks

**As a user, I want to search my tasks so that I can quickly find specific work.**

Priority: Medium

---

### US-009 Filter Tasks

**As a user, I want to filter tasks by status or category so that I can focus on relevant work.**

Priority: Medium

---

### US-010 Sort Tasks

**As a user, I want to sort tasks by priority or due date so that I can work on the most important tasks first.**

Priority: Medium

---

### US-011 Set Priority

**As a user, I want to assign priority levels so that I know which tasks are most important.**

Priority: Medium

---

### US-012 Categorize Tasks

**As a user, I want to organize tasks into categories so that similar tasks stay grouped together.**

Priority: Medium

---

### US-013 Due Dates

**As a user, I want to assign due dates so that I can complete tasks before deadlines.**

Priority: Medium

# 6. User Story Acceptance Criteria

Each user story should satisfy the following conditions:

- Provides value to the user.
- Can be independently implemented.
- Can be tested.
- Has a clear objective.
- Has defined acceptance criteria.
- Fits within a planned release.
